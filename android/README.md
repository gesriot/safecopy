# SafeCopy for Android

Нативная Kotlin-версия SafeCopy для Nothing Phone (2) и других ARM64-устройств
с Android 10 или новее. Интерфейс зафиксирован в портретной ориентации.

## Направления

- **На USB** — выбрать на смартфоне файл или папку; подключённый USB-накопитель
  обнаруживается автоматически. Системный доступ к нему выдаётся один раз.
- **С USB** — источником по умолчанию служит весь USB-накопитель. Кнопки
  **Файл** и **Папка** позволяют выбрать только часть данных. Папка назначения
  во внутренней памяти смартфона запоминается.

Выбранная папка копируется вместе со своим именем, а не только как набор её
дочерних файлов. Приложение не позволяет использовать один и тот же USB-том
одновременно как источник и назначение.

## Алгоритм

SafeCopy использует ровно один поток записи. Несколько одновременных writers на
одну флешку обычно не увеличивают пропускную способность её единственного
контроллера, но создают случайные записи, фрагментацию и дополнительный нагрев.
Хеширование и чтение выполняются последовательно с записью, чтобы проверять
именно завершённый файл и не перегружать накопитель конкурирующими запросами.

Для каждого файла приложение:

1. создаёт временный файл с точным именем и MIME `application/octet-stream`;
2. пишет данные и одновременно считает SHA-256 источника;
3. вызывает `fsync`, закрывает дескриптор и запрашивает сброс файлового кэша;
4. повторно читает временный файл с отображением прогресса и сравнивает SHA-256;
5. только после совпадения безопасно заменяет итоговый файл через backup-файл;
6. сохраняет внутреннюю контрольную точку, выдерживает паузу и после всего
   прогона повторно проверяет полную выборку.

Режим **Копировать до победного** и настройка **Без манифеста на накопителе**
включены по умолчанию. Фатальные ошибки назначения (`ENOSPC`, `EACCES`, `EROFS`,
`ENODEV` и другие) распознаются по системному errno и не попадают в бесконечный
retry. Если библиотека сброса кэша не загрузилась или `posix_fadvise` вернул
ошибку, операция останавливается: приложение больше не подменяет cold-read
молчаливым обычным чтением из page cache.

После перезапуска уже подтверждённые файлы сверяются с внутренней контрольной
точкой и пропускаются. Файл, исчерпавший попытки, получает отчёт в каталоге
`.quarantine`, после чего прогон продолжается; пять последовательных неудачных
файлов останавливают задачу. Внутренний манифест обновляется после каждого
успешного файла. Проверку копии в памяти телефона можно запустить и после
отключения USB-накопителя.

### Ограничение Android cold-read

`POSIX_FADV_DONTNEED` является подсказкой ядру, а SAF может открывать документ
через FUSE-прослойку. Поэтому на Android нельзя гарантировать обход всех уровней
кэша так же жёстко, как `FILE_FLAG_NO_BUFFERING` на Windows. SafeCopy строго
контролирует доступность собственного JNI-вызова и его результат, однако сам
эффект сброса нижележащего кэша остаётся best-effort ограничением платформы.

## Сборка

JNI-библиотека собирается Gradle-ом из `native_io.cpp`: через NDK/`ndkBuild` в
обычной Android-среде и через `scripts/build-native-arm64.sh` на ARM64-хосте.
Готовый `.so` больше не хранится как прекомпилированный исходный файл.

Debug-сборка:

```bash
gradle :app:assembleDebug
```

Подписанная release-сборка использует локальные файлы
`keystore.properties` и `signing/safecopy-release.jks`, которые исключены из
Git и требуют отдельной резервной копии:

```bash
gradle :app:assembleRelease
```

APK создаются в `app/build/outputs/apk/<variant>/` и содержат только ABI
`arm64-v8a`.
